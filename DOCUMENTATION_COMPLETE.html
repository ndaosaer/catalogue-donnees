Documentation du Catalogue de Données

Guide complet du fonctionnement de l'application Flask

Vue d'ensemble de l'application

Qu'est-ce que c'est ?

Le Catalogue de Données est une application web Flask qui transforme vos fichiers XML DDI (créés avec Nesstar Publisher) en un catalogue web moderne et interactif, inspiré du système NADA.

Objectifs principaux

Diffusion

Rendre les métadonnées des enquêtes accessibles via une interface web moderne

Recherche

Permettre la recherche rapide dans les datasets par titre, abstract et mots-clés

Documentation

Présenter les métadonnées complètes selon le standard DDI

Ressources

Donner accès aux fichiers de données, questionnaires et rapports

Technologies utilisées

Composant Technologie Rôle
Backend Python Flask 3.x Serveur web et routage
Parser XML lxml Extraction des métadonnées DDI
Templates Jinja2 Génération HTML dynamique
Frontend HTML5, CSS3, JavaScript Interface utilisateur
Ressources JSON Configuration des ressources externes

Fonctionnalités clés

  • Extraction automatique des métadonnées depuis les fichiers XML DDI
  • Catalogue complet avec liste de tous les datasets
  • Pages détaillées pour chaque enquête avec 5 onglets thématiques
  • Recherche en temps réel avec suggestions instantanées
  • Ressources externes : liens vers données, questionnaires, rapports
  • Design responsive adapté mobile, tablette et desktop
  • Page À propos avec informations sur le catalogue

Rôle de chaque fichier/dossier

app.py - Le cerveau de l'application

  • Charge et parse les fichiers XML DDI
  • Définit les routes (URLs) de l'application
  • Gère les requêtes HTTP et génère les réponses
  • Charge les ressources externes depuis resources.json

data/ - Vos données sources

Placez tous vos fichiers XML DDI ici. L'application les charge automatiquement au démarrage. Chaque fichier .xml devient un dataset dans le catalogue.

templates/ - Les pages HTML

  • base.html : Template parent avec header, nav, footer
  • index.html : Page d'accueil avec recherche et stats
  • catalog.html : Liste de tous les datasets
  • dataset.html : Page détail avec 5 onglets
  • about.html : Informations sur le catalogue

resources.json - Configuration ressources

Fichier JSON où vous configurez les liens vers vos fichiers de données, questionnaires, rapports et documentation pour chaque dataset.

Routes de l'application

Route Méthode Description
/ index() Page d'accueil avec recherche et statistiques
/catalog catalog() Liste complète des datasets
/dataset/<id> dataset_detail(id) Page détaillée d'un dataset spécifique
/about about() Page À propos du catalogue
/search?q=... search() API de recherche (retourne JSON)

Comment ça marche ?

Quand un utilisateur visite une page (ex: /catalog), Flask :

  1. Appelle la fonction correspondante (catalog())
  2. Prépare les données (liste des datasets)
  3. Rend le template HTML (catalog.html) avec ces données
  4. Envoie le HTML généré au navigateur

Fonctionnement du Frontend

Templates Jinja2

Les templates utilisent la syntaxe Jinja2 pour insérer dynamiquement les données dans le HTML.

<!-- Exemple dans index.html --> <h1>Bienvenue</h1> <!-- Boucle sur tous les datasets --> {% for dataset_id, dataset in datasets.items() %} <div class="dataset-card"> <h3>{{ dataset.metadata.title }}</h3> <p>{{ dataset.metadata.abstract }}</p> </div> {% endfor %}

Système d'onglets (JavaScript)

Sur la page détail d'un dataset, les onglets utilisent jQuery :

// Quand on clique sur un onglet $('.tab').click(function() { var tabId = $(this).data('tab'); // 1. Retire la classe "active" de tous les onglets $('.tab').removeClass('active'); $('.tab-content').removeClass('active'); // 2. Ajoute "active" à l'onglet cliqué $(this).addClass('active'); $('#' + tabId).addClass('active'); });

Recherche en temps réel

Sur la page d'accueil, la recherche envoie des requêtes AJAX :

// Quand l'utilisateur tape dans la barre de recherche $('#search-input').on('input', function() { var query = $(this).val(); // Envoie une requête à /search $.get('/search', { q: query }, function(data) { // Affiche les résultats afficherResultats(data.results); }); });

Styles CSS

Le fichier style.css utilise des variables CSS pour faciliter la personnalisation :

:root { --primary-color: #2563eb; --secondary-color: #64748b; --dark-color: #1e293b; /* ... */ } .btn-primary { background: var(--primary-color); color: white; padding: 0.75rem 1.5rem; border-radius: 8px; }

Design responsive

Le CSS utilise des media queries et des grids flexibles pour adapter l'affichage automatiquement sur mobile, tablette et desktop.

Cycle de vie d'une requête

Exemple : Utilisateur visite /catalog

  1. Navigateur → Envoie requête GET à http://localhost:5001/catalog
  2. Flask → Reçoit la requête et appelle catalog()
  3. catalog() → Récupère DATASETS et EXTERNAL_RESOURCES
  4. Jinja2 → Génère catalog.html avec les données
  5. Flask → Envoie le HTML au navigateur
  6. Navigateur → Affiche la page, charge CSS et JS
  7. JavaScript → Active les fonctionnalités interactives

Flux des ressources externes

1. Au démarrage : load_external_resources() └─ Lit resources.json └─ Stocke dans EXTERNAL_RESOURCES 2. Sur page détail (/dataset/ID) : dataset_detail(ID) └─ Récupère metadata de DATASETS[ID] └─ Récupère resources de EXTERNAL_RESOURCES[ID] └─ Passe les deux au template dataset.html 3. Template affiche : Onglet "Ressources externes" └─ Boucle sur data_files, questionnaires, etc. └─ Affiche cartes cliquables avec liens

Important : Ordre de chargement

Les données sont chargées une seule fois au démarrage. Si vous modifiez les fichiers XML ou resources.json, vous devez redémarrer l'application pour voir les changements.

Gestion des ressources externes

Qu'est-ce qu'une ressource externe ?

Ce sont des fichiers complémentaires que vous voulez rendre accessibles aux utilisateurs du catalogue :

Fichiers de données

CSV, SPSS, Stata, Excel

Questionnaires

Instruments de collecte, manuels

Rapports

Publications, synthèses

Documentation

Guides, codebooks, protocoles

Structure du fichier resources.json

{ "BEN-Afrobarometer-EOP-2025-001": { "data_files": [ { "name": "Données SPSS", "description": "Fichier au format SPSS", "type": "spss", "icon": "📊", "url": "https://exemple.com/data.sav", "size": "2.5 MB" } ], "questionnaires": [ /* ... */ ], "reports": [ /* ... */ ], "documentation": [ /* ... */ ] } }

Comment ajouter vos ressources

  1. Ouvrez resources.json dans un éditeur de texte
  2. Trouvez l'ID de votre dataset (visible dans l'URL ou le XML)
  3. Ajoutez vos ressources en suivant le format ci-dessus
  4. Hébergez vos fichiers sur un serveur web
  5. Mettez les URLs dans le fichier JSON
  6. Redémarrez l'application

Astuce : Hébergement des fichiers

Vous pouvez héberger vos fichiers sur :

  • Votre site web institutionnel
  • Google Drive (avec lien de partage)
  • Dropbox, OneDrive
  • Serveur dédié

Affichage dans le catalogue

Les ressources apparaissent à 2 endroits :

1. Page Catalogue (/catalog)

Sous chaque dataset, un aperçu affiche le nombre de ressources disponibles :

📚 Ressources disponibles 📊 3 fichier(s) de données 📝 5 questionnaire(s) 📄 4 rapport(s) 📑 4 document(s) technique(s)

2. Onglet Ressources externes (/dataset/ID)

Un onglet complet avec des cartes élégantes pour chaque ressource, organisées par catégorie. Chaque carte affiche :

  • Icône distinctive
  • Nom de la ressource
  • Description
  • Type et taille du fichier
  • Bouton de téléchargement
