📡 1. Introduction à l'API

Qu'est-ce que l'API du catalogue ?

L'API (Application Programming Interface) du catalogue est un point d'accès HTTP qui permet de rechercher des datasets en envoyant une requête et en recevant les résultats au format JSON.

À quoi sert cette API ?

🔍

Recherche dynamique

Permet à l'interface web d'afficher des résultats de recherche en temps réel

⚡

Performance

Évite de recharger toute la page, seuls les résultats sont mis à jour

🔗

Intégration

Peut être utilisée par d'autres applications pour interroger le catalogue

📊

Format structuré

Retourne des données JSON faciles à traiter programmatiquement

💡 En résumé

Quand vous tapez dans la barre de recherche de la page d'accueil et que les résultats apparaissent instantanément, c'est grâce à cette API qui travaille en arrière-plan !

🤔 2. Qu'est-ce qu'une API ? (Explication simple)

Analogie du restaurant

👤 Le client (Navigateur web)

Vous êtes dans un restaurant et vous voulez manger. Vous ne pouvez pas aller directement en cuisine.

📋 Le serveur (API)

Le serveur prend votre commande, va en cuisine, et vous rapporte votre plat. C'est l'intermédiaire.

👨‍🍳 La cuisine (Backend Flask)

Le chef (Flask) prépare votre plat selon votre commande et le donne au serveur.

🍽️ Le plat (Réponse JSON)

Vous recevez exactement ce que vous avez commandé, prêt à être consommé.

Dans notre catalogue

Élément du restaurant Équivalent dans notre API
Le client Votre navigateur web (Chrome, Firefox, etc.)
La commande Requête HTTP GET avec un terme de recherche (ex: "MICS")
Le serveur L'endpoint API /search
La cuisine La fonction Python search() dans app.py
Le plat Réponse JSON avec les datasets trouvés

🎯 Pourquoi utiliser une API ?

Sans API, il faudrait recharger toute la page à chaque recherche. Avec l'API, seuls les résultats sont mis à jour, ce qui est beaucoup plus rapide et fluide !

🎯 3. L'endpoint de recherche

GET Recherche de datasets
http://localhost:5001/search

Permet de rechercher des datasets par mot-clé dans le catalogue

Qu'est-ce qu'un endpoint ?

Un endpoint (point de terminaison) est une URL spécifique qui accepte des requêtes HTTP et retourne des données. C'est comme une adresse postale : vous envoyez votre lettre (requête) à cette adresse, et vous recevez une réponse.

Décomposition de l'URL

Partie Valeur Signification
Protocole http:// Type de communication (HTTP)
Hôte localhost Serveur local (votre ordinateur)
Port 5001 Port sur lequel Flask écoute
Chemin /search Route de recherche

⚠️ En production

En production (site web déployé), l'URL serait :
https://monsite.com/search
Au lieu de http://localhost:5001/search

⚙️ 4. Paramètres de la requête

Paramètre unique : q

Nom Type Requis ? Description
q string Optionnel Le terme de recherche (mot-clé à chercher dans les datasets)

Comment passer le paramètre ?

Format de l'URL avec paramètre

http://localhost:5001/search?q=votre_recherche

Le ? indique le début des paramètres.
Ensuite, on a nom_paramètre=valeur

Exemples d'URLs complètes

Exemple 1 : Rechercher "MICS"

http://localhost:5001/search?q=MICS

Exemple 2 : Rechercher "démocratie"

http://localhost:5001/search?q=démocratie

Exemple 3 : Rechercher "Bénin 2024"

http://localhost:5001/search?q=Bénin 2024

Note: Les espaces sont automatiquement convertis en %20 dans l'URL

⚠️ Si aucun paramètre n'est fourni

Si vous appelez /search sans paramètre q ou avec q vide, l'API retourne un tableau vide :

{ "results": [] }

📦 5. Format de la réponse

Type de réponse

L'API retourne toujours un objet JSON avec une clé results contenant un tableau de datasets trouvés.

Structure de la réponse

{ "results": [ { "id": "ID_DU_DATASET", "title": "Titre du dataset", "abstract": "Résumé (max 200 caractères)..." }, // ... autres résultats ] }

Champs retournés pour chaque résultat

Champ Type Description
id string Identifiant unique du dataset (ex: "BEN-Afrobarometer-EOP-2025-001")
title string Titre complet de l'enquête
abstract string Résumé tronqué à 200 caractères + "..." si nécessaire

Exemple de réponse réelle

Requête : /search?q=MICS

{ "results": [ { "id": "DDI-BEN-INS-MICS-2014-V2.0", "title": "Bénin - Enquête par Grappe à Indicateurs Multiples MICS 2021-2022", "abstract": "Le programme mondial des enquêtes MICS a été élaboré par l'UNICEF dans les années 1990 en tant que programme international d'enquêtes auprès des ménages qui vise à soutenir les pays dans la collecte..." } ] }

Requête : /search?q=xyz (aucun résultat)

{ "results": [] }

💻 6. Exemples d'utilisation

Exemple 1 : Tester dans le navigateur

📌 Le plus simple !

Ouvrez votre navigateur et tapez directement l'URL :

http://localhost:5001/search?q=MICS

Le navigateur affichera le JSON retourné par l'API !

Exemple 2 : Utiliser curl (Terminal)

# Rechercher "démocratie" curl "http://localhost:5001/search?q=démocratie" # Avec formatage JSON (jq installé) curl "http://localhost:5001/search?q=MICS" | jq

Exemple 3 : JavaScript (AJAX avec jQuery)

// Recherche avec jQuery $.get('/search', { q: 'MICS' }, function(data) { console.log(data.results); // Afficher les résultats data.results.forEach(function(result) { console.log(result.title); }); });

Exemple 4 : JavaScript moderne (Fetch API)

// Recherche avec fetch async function searchDatasets(query) { const response = await fetch(`/search?q=${query}`); const data = await response.json(); return data.results; } // Utilisation searchDatasets('Afrobarometer') .then(results => { console.log(`Trouvé ${results.length} résultat(s)`); });

Exemple 5 : Python (requests)

import requests # Rechercher dans le catalogue response = requests.get( 'http://localhost:5001/search', params={'q': 'MICS'} ) data = response.json() results = data['results'] for dataset in results: print(f"Titre: {dataset['title']}")

⚙️ 7. Comment ça fonctionne (Backend)

Code de la fonction search() dans app.py

@app.route('/search') def search(): """Recherche dans les datasets""" # 1. Récupérer le paramètre 'q' de l'URL query = request.args.get('q', '').lower() # 2. Si pas de query, retourner vide if not query: return jsonify({'results': []}) results = [] # 3. Parcourir tous les datasets for dataset_id, dataset in DATASETS.items(): metadata = dataset['metadata'] # 4. Construire le texte de recherche # (titre + abstract + mots-clés) searchable = f"{metadata['title']} {metadata['abstract']} " searchable += ' '.join(metadata['keywords']) searchable = searchable.lower() # 5. Vérifier si le terme est présent if query in searchable: # 6. Tronquer l'abstract à 200 caractères abstract = metadata['abstract'] if len(abstract) > 200: abstract = abstract[:200] + '...' # 7. Ajouter aux résultats results.append({ 'id': dataset_id, 'title': metadata['title'], 'abstract': abstract }) # 8. Retourner les résultats en JSON return jsonify({'results': results})

Étapes détaillées du fonctionnement

  1. Réception de la requête

    Flask reçoit la requête HTTP GET sur /search

  2. Extraction du paramètre

    request.args.get('q', '') récupère le terme de recherche de l'URL

  3. Normalisation

    Le terme est converti en minuscules avec .lower() pour une recherche insensible à la casse

  4. Vérification

    Si le paramètre est vide, retourne immédiatement un tableau vide

  5. Parcours des datasets

    Boucle sur tous les datasets chargés dans DATASETS

  6. Construction du texte searchable

    Concatène titre + abstract + mots-clés pour créer un texte de recherche

  7. Recherche

    Vérifie si le terme de recherche est présent dans le texte (avec in)

  8. Formatage des résultats

    Pour chaque match, crée un objet avec id, title et abstract tronqué

  9. Sérialisation JSON

    jsonify() convertit le dictionnaire Python en JSON

  10. Envoi de la réponse

    Flask envoie le JSON au client avec le header Content-Type: application/json

🔍 Type de recherche

La recherche actuelle est simple : elle cherche si le terme est contenu dans le texte (recherche par substring). Par exemple, chercher "MICS" trouvera aussi "MICS Bénin" ou "enquête MICS".

🎨 8. Utilisation côté Frontend

Code JavaScript dans index.html

$(document).ready(function() { // Variable pour le délai de recherche let searchTimeout; // Écouter les changements dans le champ de recherche $('#search-input').on('input', function() { // Annuler la recherche précédente clearTimeout(searchTimeout); // Récupérer le terme saisi const query = $(this).val().trim(); // Si moins de 3 caractères, masquer les résultats if (query.length < 3) { $('#search-results').hide(); return; } // Attendre 300ms avant de lancer la recherche // (pour éviter trop de requêtes pendant la frappe) searchTimeout = setTimeout(function() { // Appel AJAX à l'API $.get('/search', { q: query }, function(data) { if (data.results.length > 0) { // Construire le HTML des résultats let html = '<div class="results-list">'; data.results.forEach(function(result) { html += ` <div class="result-item"> <a href="/dataset/${result.id}"> <h4>${result.title}</h4> <p>${result.abstract}</p> </a> </div> `; }); html += '</div>'; // Afficher les résultats $('#search-results').html(html).show(); } else { // Aucun résultat $('#search-results') .html('<div class="no-results">Aucun résultat trouvé</div>') .show(); } }); }, 300); // Délai de 300ms }); // Fermer les résultats en cliquant ailleurs $(document).on('click', function(e) { if (!$(e.target).closest('.search-container').length) { $('#search-results').hide(); } }); });

Flux de l'interaction utilisateur

1. Utilisateur tape dans la barre de recherche

L'événement input est déclenché à chaque caractère tapé

2. JavaScript attend 300ms

Pour éviter d'envoyer une requête à chaque touche, on attend que l'utilisateur finisse de taper

3. Vérification de la longueur (min 3 caractères)

Si moins de 3 caractères, la recherche ne se lance pas

4. Requête AJAX envoyée à /search

Le navigateur envoie la requête en arrière-plan (sans recharger la page)

5. Backend traite la recherche

Flask parcourt les datasets et retourne les résultats en JSON

6. JavaScript reçoit la réponse

La fonction callback est exécutée avec data contenant les résultats

7. Construction du HTML

Pour chaque résultat, on crée une carte HTML avec titre et abstract

8. Affichage des résultats

Les résultats sont insérés dans #search-results et affichés

✨ Avantages de cette approche

  • Instantané : Les résultats apparaissent sans recharger la page
  • Fluide : L'utilisateur continue de taper pendant la recherche
  • Optimisé : Le délai de 300ms évite trop de requêtes
  • Ergonomique : Les résultats se ferment automatiquement

🎯 9. Cas d'usage pratiques

Cas 1 : Recherche interactive sur la page d'accueil

Situation : L'utilisateur veut trouver rapidement une enquête

Solution : Barre de recherche avec résultats instantanés via l'API

Avantage : Expérience utilisateur fluide et rapide

Cas 2 : Filtrage dans la page catalogue

Situation : Sur /catalog, filtrer les datasets affichés

Solution : Formulaire qui appelle l'API et met à jour la liste

Avantage : Pas besoin de recharger toute la page

Cas 3 : Intégration dans un autre site

// Un autre site web peut utiliser votre API fetch('http://votre-catalogue.com/search?q=santé') .then(res => res.json()) .then(data => { // Afficher les datasets sur leur propre site afficherDatasets(data.results); });

Cas 4 : Application mobile

Situation : Créer une app mobile pour consulter le catalogue

Solution : L'app envoie des requêtes à l'API et affiche les résultats

Avantage : Même API pour web et mobile

Cas 5 : Script d'analyse

# Script Python pour analyser le catalogue import requests def chercher_enquetes_par_theme(theme): response = requests.get( 'http://localhost:5001/search', params={'q': theme} ) return response.json()['results'] # Trouver toutes les enquêtes sur la santé enquetes_sante = chercher_enquetes_par_theme('santé') print(f"Trouvé {len(enquetes_sante)} enquête(s) sur la santé")

💡 Extensibilité

Cette API simple peut être étendue avec d'autres endpoints :

  • /api/datasets - Lister tous les datasets
  • /api/dataset/<id> - Obtenir un dataset spécifique
  • /api/statistics - Obtenir des statistiques du catalogue
  • /api/filters - Filtrer par année, producteur, etc.
