📦 Étape 1 : Mettre le code sur GitHub

📁 Structure de votre dossier avant de commencer

Assurez-vous que votre dossier Catalogue_Donnees_Flask contient ces fichiers :

  • app.py, requirements.txt, render.yaml, .gitignore
  • templates/ (base.html, index.html, catalog.html, dataset.html, about.html)
  • static/css/style.css
  • data/ (vos fichiers XML)
  • resources.json
1

Ouvrir le terminal dans votre dossier

Dans l'explorateur Windows, naviguez vers votre dossier Catalogue_Donnees_Flask, puis :

Clic droit dans le dossier → "Ouvrir dans le terminal" (ou Git Bash)

# Vérifier que vous êtes dans le bon dossier pwd # Doit afficher quelque chose comme : C:/Users/Easy Services Pro/Desktop/AS3_2025_2026/Projet_archivage/Catalogue_Donnees_Flask
2

Initialiser Git dans le dossier

Cette commande crée un dépôt Git local dans votre dossier :

git init # Réponse attendue : Initialized empty Git repository in ...
3

Ajouter tous les fichiers

Le point (.) signifie "tous les fichiers du dossier courant" :

git add . # Pas de message = c'est normal, tout s'est bien passé
4

Faire le premier commit

Un commit est une "sauvegarde" de l'état actuel de votre code :

git commit -m "Premier commit - Catalogue de données v1.0" # Réponse : [master (root-commit) xxxxx] Premier commit...
5

Créer un dépôt sur GitHub.com

Allez sur github.com et :

  • Cliquez sur le bouton vert "New" (ou "+" → "New repository")
  • Nom du dépôt : catalogue-donnees
  • Laissez en Public
  • Ne cochez pas "Add a README file"
  • Cliquez "Create repository"

⚠️ Important

Ne cochez aucune case optionnelle, votre dépôt doit être vide pour recevoir votre code.

6

Connecter votre dossier à GitHub

GitHub vous affichera des commandes après la création. Utilisez celles-ci :

# Remplacez VOTRE_USERNAME par votre nom d'utilisateur GitHub git remote add origin https://github.com/VOTRE_USERNAME/catalogue-donnees.git git branch -M main
7

Envoyer le code sur GitHub

git push -u origin main # GitHub demandera votre identifiant + mot de passe (ou token) # Réponse finale attendue : # Branch 'main' set up to track remote branch 'main' from 'origin'.

✅ Vérification

Allez sur github.com/VOTRE_USERNAME/catalogue-donnees. Vous devez voir tous vos fichiers !

🌐 Étape 2 : Déployer sur Render

✅ Prérequis

Votre code doit être sur GitHub avant de commencer cette étape.

1

Créer un compte Render

  • Allez sur render.com
  • Cliquez "Get Started for Free"
  • Choisissez "Sign up with GitHub" ← plus simple !
  • Autorisez Render à accéder à votre GitHub

💡 Pourquoi se connecter avec GitHub ?

Render peut ainsi lire directement vos dépôts GitHub sans configuration supplémentaire.

2

Créer un nouveau Web Service

  • Dans le dashboard Render, cliquez "New +"
  • Sélectionnez "Web Service"
  • Dans "Connect a repository", trouvez catalogue-donnees
  • Cliquez "Connect"
3

Configurer le service

Render va vous demander de remplir ces champs :

ChampValeur à saisir
Namecatalogue-donnees
RegionFrankfurt (EU Central) — le plus proche d'Afrique
Branchmain
RuntimePython 3
Build Commandpip install -r requirements.txt
Start Commandgunicorn app:app
Instance TypeFree

⚠️ Build Command et Start Command

Ces deux champs sont les plus importants. Copiez-les exactement comme indiqué. Si Render les remplit automatiquement grâce au render.yaml, vérifiez qu'ils correspondent.

4

Lancer le déploiement

  • Cliquez "Create Web Service"
  • Render va afficher les logs en direct
  • Attendez de voir : "Your service is live 🎉"
  • Durée : environ 2-5 minutes
# Vous verrez dans les logs : ==> Cloning from https://github.com/VOTRE_USERNAME/catalogue-donnees ==> Running build command: pip install -r requirements.txt ==> Installing Flask, lxml, gunicorn... ==> Running start command: gunicorn app:app 📂 Chargement des datasets depuis: /opt/render/project/data ✅ 2 dataset(s) chargé(s) avec succès! Listening on http://0.0.0.0:10000
5

Récupérer votre URL publique

En haut du dashboard Render, vous verrez votre URL :

https://catalogue-donnees.onrender.com

🎉 Votre site est en ligne !

Partagez cette URL à n'importe qui dans le monde pour qu'ils puissent consulter votre catalogue.

⚠️ Limite du plan gratuit Render

Mise en veille après 15 minutes d'inactivité

Sur le plan gratuit, si personne ne visite le site pendant 15 minutes, il se "met en veille". La prochaine visite prend 30-60 secondes pour "réveiller" le serveur. C'est normal et gratuit. Pour éviter ça, il faut passer au plan payant ($7/mois) ou utiliser un service de "ping" gratuit.

🔄 Mettre à jour le catalogue après déploiement

C'est la vraie magie de Git + Render : une modification = 3 commandes = site mis à jour ✨

Le workflow de mise à jour

✏️

Modifier votre code

Sur votre PC
→
💾

git add + commit

Sauvegarder
→
📤

git push

Envoyer
→
🚀

Render redéploie

Automatique !

Les 3 commandes magiques (à retenir !)

# Après avoir modifié un fichier quelconque # 1. Ajouter les modifications git add . # 2. Décrire ce que vous avez changé git commit -m "Description de votre modification" # 3. Envoyer sur GitHub → Render se met à jour automatiquement git push

Cas pratiques de mises à jour

A

Ajouter une nouvelle enquête XML

# 1. Copiez votre nouveau fichier XML dans data/ # 2. Puis : git add . git commit -m "Ajout enquête EDS 2023" git push # Render redémarre et charge automatiquement le nouveau dataset
B

Modifier les ressources externes (resources.json)

# 1. Ouvrez resources.json et ajoutez vos nouveaux liens # 2. Sauvegardez le fichier # 3. Puis : git add resources.json git commit -m "Mise à jour des ressources MICS" git push
C

Modifier le design (style.css) ou les templates

# 1. Modifiez static/css/style.css ou templates/*.html # 2. Puis : git add . git commit -m "Amélioration du design de la page d'accueil" git push
D

Modifier le titre ou les informations du site

# 1. Modifiez templates/base.html pour changer le titre # 2. Puis les 3 commandes habituelles : git add . git commit -m "Changement du titre du catalogue" git push

Revenir à une version précédente

🕐 En cas de problème

Si une mise à jour cause un problème, vous pouvez revenir en arrière :

# Voir l'historique des commits git log --oneline # Affiche quelque chose comme : # abc1234 Ajout enquête EDS 2023 # def5678 Mise à jour ressources MICS # ghi9012 Premier commit # Revenir au commit précédent (remplacez par votre hash) git revert abc1234 git push # Le site revient automatiquement à l'état précédent

💡 Bonne pratique

Décrivez toujours clairement vos commits : "Ajout enquête RGPH 2023" plutôt que "modif". Ça vous aidera à retrouver facilement une version précise.

Résumé des commandes Git essentielles

CommandeQuand l'utiliser
git add . Avant chaque commit, pour inclure tous les fichiers modifiés
git commit -m "message" Pour sauvegarder une version de votre code avec une description
git push Pour envoyer sur GitHub et déclencher le redéploiement sur Render
git status Pour voir quels fichiers ont été modifiés
git log --oneline Pour voir l'historique de vos commits
git pull Pour récupérer les dernières modifications depuis GitHub
